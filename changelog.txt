## Version 2.1.81 (Latest)
- Pinned managed AsaApi installations to checksum-verified version 2.01 while allowing explicit, removable `AsaApi_Custom` overrides for advanced users.
- Bypassed AsaApi's failing Wine/OpenSSL HTTPS cache path by downloading executable-specific caches with native Linux `curl`, validating their archive and serialized-map structure, and installing generation metadata atomically.
- Added a cross-container AsaApi lock so simultaneous multi-instance starts reuse one verified release/cache, plus safe offline reuse when an existing matching cache is valid.
- API-enabled servers now wait and retry instead of silently launching without plugins, expose the cache wait through container health, and require `API was successfully loaded` before managed API startup is considered healthy.
- Added an atomic two-stage shutdown barrier that independently verifies both `SaveWorld` and the save performed by `DoExit` before Docker may stop a container.
- `-stop -all`, `-shutdown`, and `-restart` now dispatch each save stage to all targeted instances concurrently; any failed instance aborts the whole operation before containers are removed.
- Added a trailing `--force` escape hatch with prominent data-loss warnings, reliable RCON failure propagation, fresh log checkpoints, and stable main `.ark` metadata fallback based on each instance's unrestricted `MAP_NAME`.
- Increased the new-instance `SAVE_WAIT_SECONDS` default to 60 seconds and range to 1-900 while preserving existing values, including 5-second configurations.
- Added derived Compose/Docker grace periods, clean PID 1 SIGTERM handling, and verified-save protection across automatic update, restart, recovery, delete, and migration paths.
